# ISO / IEC / ITU Standards Terminology Boundary Note ｜ ISO / IEC / ITU 標準化周辺用語境界メモ

## Purpose ｜ 目的

This note provides standards-adjacent terminology for explaining LUMINA-30 near AI standards, assurance, audit, and governance discussions.
It does not claim that LUMINA-30 is an ISO, IEC, ITU, IEEE, national, sectoral, or institutional standard.

本メモは、AI標準、保証、監査、ガバナンス議論の周辺でLUMINA-30を説明するための用語整理である。
LUMINA-30がISO、IEC、ITU、IEEE、国家、分野別、または組織内標準であると主張しない。

## Status ｜ 位置づけ

This note is non-binding, descriptive, and unaffiliated with any external institution.
It does not claim endorsement, adoption, recognition, approval, certification, compliance, legal effect, or institutional use by any external organization.

本メモは、非拘束的・記述的であり、いかなる外部機関とも提携していない。
外部機関による承認、採用、認定、承諾、認証、準拠、法的効力、または制度利用を主張しない。

- a certification scheme
- a compliance framework
- an implementation requirement
- a safety guarantee
- an official assurance method
- a replacement for ISO / IEC / ITU standards

## Terminology Bridge ｜ 用語橋渡し

| Existing discussion | LUMINA-30 boundary wording |
|---|---|
| Human oversight | Was oversight strong enough to preserve effective refusal? |
| Accountability | Can responsibility for refusal, stop, delay, or override be traced? |
| Auditability | Can the refusal pathway be verified by records? |
| Incident response | Before containment or recovery, was refusal still exercisable? |
| AI assurance | Does assurance include a pre-irreversibility refusal check? |

## Attachment Text ｜ 添付用短文

LUMINA-30 should not be presented as a standard or compliance framework.
It can be used as standards-adjacent terminology for a supplementary boundary question: whether formal human oversight remained operationally sufficient to preserve effective refusal before irreversible AI-related impact.

LUMINA-30は、標準または準拠フレームワークとして提示してはならない。
標準化周辺の用語として、形式的人間監督が不可逆的AI関連影響の前に実効的拒否を保全できるほど運用上十分だったか、という補助的境界問いに使用できる。
