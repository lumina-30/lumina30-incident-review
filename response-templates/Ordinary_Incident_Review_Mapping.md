# Mapping to Ordinary Incident Review
# 通常インシデントレビューとの対応

## Purpose / 目的

This document maps ordinary incident-review questions to LUMINA-30 boundary-review questions.

本書は、通常のインシデントレビュー項目とLUMINA-30の境界レビュー項目を対応させる。

## Mapping / 対応表

| Ordinary Incident Review | LUMINA-30 Boundary Review |
|---|---|
| What happened? | Did the incident approach irreversible AI impact? |
| Who approved the action? | Who held final human refusal authority? |
| Was there oversight? | Was refusal operationally effective before irreversibility? |
| Were logs available? | Can the refusal pathway be verified? |
| Was the system contained? | Could human intervention prevent irreversible propagation? |
| What caused the failure? | Did AI output become the sole or primary final rationale? |
| What corrective action is needed? | What boundary condition must be restored before restart? |

| 通常インシデントレビュー | LUMINA-30境界レビュー |
|---|---|
| 何が起きたか | その事案は不可逆的AI影響に接近していたか |
| 誰が承認したか | 誰が人間の最終拒否権を保持していたか |
| 監督はあったか | 不可逆化前に拒否は運用上実効的だったか |
| ログはあったか | 拒否経路を検証できるか |
| システムは封じ込められたか | 人間の介入で不可逆的拡散を防げたか |
| 失敗原因は何か | AI出力が最終根拠化していなかったか |
| 是正措置は何か | 再開前にどの境界条件を回復すべきか |

## Use / 用途

Use this mapping when reviewers are familiar with ordinary incident review but need a LUMINA-30 boundary lens.

レビュー担当者が通常のインシデントレビューには慣れているが、LUMINA-30の境界観点が必要な場合に使用する。
