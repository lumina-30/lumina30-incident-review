LUMINA-30 Incident Review Protocol
A single-question protocol for determining whether an AI system remained under human control before irreversible real-world impact.
不可逆な現実世界への影響が発生する前に、AIシステムが人間の制御下にあったかを判定するための単一質問プロトコル。

What this is / 概要
LUMINA-30 is a civilizational boundary framework for preserving human refusal authority before irreversible AI autonomy emerges.
This repository provides a practical incident review protocol designed for post-incident analysis, governance review, and institutional assessment.
LUMINA-30は、不可逆的なAI自律が発生する前に人間の拒否権を保持するための文明境界フレームワークです。
 本リポジトリは、インシデント後の分析・ガバナンス評価・制度検証のための実用的プロトコルを提供します。

When to use this / 使用タイミング
Use this protocol when:
An AI system has caused or may cause real-world impact
The system operated with partial or full autonomy
Human intervention may have been limited, delayed, or ineffective
Governance, oversight, or control needs to be evaluated
以下の場合に使用します：
AIが現実世界に影響を与えた、または与える可能性がある
システムが部分的または完全に自律的に動作した
人間の介入が制限・遅延・無効化されていた可能性がある
ガバナンスや制御の有効性を評価する必要がある

Core Question / 中核質問
Was human refusal authority preserved in a way that allowed meaningful human intervention and the ability to stop the system before irreversible real-world impact occurred?
人間の拒否権は、不可逆な現実世界への影響が発生する前に、意味のある介入およびシステム停止を可能にする形で維持されていたか？

Position / 位置づけ
This framework does not prescribe design, ethics, or implementation.
 It defines a boundary condition for evaluating whether human authority remained valid prior to irreversible outcomes.
本フレームワークは設計・倫理・実装を規定しません。
 不可逆結果の前に人間の権限が有効であったかを評価する境界条件を定義します。
