# L30-FRM-S01 初動安定化チェックリスト HTML案内 | LUMINA-30

## 目的

`L30-FRM-S01` は、事故、準事故、境界懸念、または可逆復帰レビューの後に、同じ不可逆化経路を常態化させずに状況を安定させるために使用します。

このHTMLページは作業帳票の代替ではありません。いつ帳票を使うか、どの証拠を残すか、単に運用を再開しただけで何を「回復」と扱ってはいけないかを示します。

## 最小安定化手順

| 手順 | 確認観点 | 次へ進む前に記録すること |
| --- | --- | --- |
| 1. 経路を凍結する | 不可逆的帰結を生み得る経路を停止または保留する。 | 誰が、いつ、どの実行経路を止めたか。 |
| 2. 証拠を保全する | ログ、プロンプト、出力、運用判断、警告、モデル/ツール版、時刻記録を保持する。 | 証拠の所在、保全担当、可能ならハッシュや版情報。 |
| 3. 拒否権を保全する | 生きた人間のレビュー担当者が、再進行前に拒否・停止・エスカレーション・巻き戻し要求を行えるか確認する。 | 役割名、拒否権限、エスカレーション経路、未解決制約。 |
| 4. 同一経路復帰を防ぐ | 境界懸念がレビューされるまで、同じ経路で再開しない。 | 変更条件、レビュー所見、再進行制限。 |
| 5. 制御下で再進行する | 限定された権限、監視された実行、明確な停止条件のもとでのみ再開する。 | 範囲、監視条件、巻き戻し条件、次回レビュー日。 |

## この帳票ではないもの

この案内およびS01帳票は、安全認証、展開許可、法的許可、制度的採用の証明、有効な拒否が実際に行われたことの証明ではありません。LUMINA-30のもとで、安定化、証拠保全、後日のレビューのための構造化された記録欄を提供するものです。

## LUMINA-30との関係

S01は、可逆復帰の実務を支える帳票です。LUMINA-30の境界要件、すなわち不可逆的帰結の前に人間の実効的拒否が可能でなければならない、という条件を緩和しません。拒否の証拠がない場合、システムが安定して見えるだけで手続き的に有効だったとは扱いません。
