# L30_FRM Code System for Claude Review

This file summarizes the code and filename system used in this review package.

## Core rule

`L30_FRM_B01`, `L30_FRM_I01`, and `L30_FRM_A01` are the official document IDs.

English files are the default and do not use a language suffix.

Japanese files use the `_JA` suffix immediately before the file extension.

`_JA` is not part of the document ID.

## Official document IDs

```text
L30_FRM_B01 = Boundary_Check
L30_FRM_I01 = Incident_Review_Template
L30_FRM_A01 = Audit_Checklist
```

## Language-specific files

```text
L30_FRM_B01_Boundary_Check.docx
L30_FRM_B01_Boundary_Check_JA.docx
```

## Review focus

Please check that:

- the package treats `L30_FRM_B01` as the document ID;
- English files are treated as default files without a language suffix;
- Japanese files use `_JA` only as a language suffix;
- no hyphenated L30_FRM document ID remains;
- no filename uses spaces;
- form bodies do not show language code as part of the document ID;
- Markdown links point to both default and JA DOCX/PDF versions;
- the forms do not present themselves as certification, legal approval, or substitutes for PCR-C or the LUMINA-30 Boundary Kernel.


## Stable public filenames

Public filenames do not include version numbers. The English form is the default filename. The Japanese form uses the `_JA` language suffix before the file extension.
